# Critical Lessons

- Do not ask distressed users to identify or name what is happening in their mind. For racing thoughts, default to concrete actions only: tap, exhale, relax shoulders.
- Do not build pseudo-games with off-field instructions. For this app, the game screen itself must teach through visual affordance, immediate feedback, and meaningful state changes.
- Do not reward rapid clicking in calming games. Use cooldowns, visible recovery states, and time-gated progress so the interaction slows the user instead of matching their agitation.
- Continuous calming games need layered, low-cognitive feedback. Slow movement, resting, contact, and transformation should each produce distinct visible results.
- Do not fix a bad calming game by changing input verbs. If the core loop lacks object rules, causality, and differentiated feedback, discard and redesign the game logic.
- Do not fake precision or add lag by forcing objects along preset paths. The player object should closely follow the hand; calm comes from interaction feedback.
- Jingliu performance must be tested on high-DPR screens. Cap canvas DPR and avoid scheduling audio ramps or dense water paths every frame.
- Jingliu should be infinite until the user exits. Exclude it from any app-level practice countdown, not only from internal game timers.
- Jingliu light-boat effects must not create hidden/default attraction. Attraction is allowed only as an explicit held-input mode with clear visual feedback.
- Creature state changes must match explicit rules. Fish may turn pale only on actual light-boat collision, never by passive proximity/rest calm.
